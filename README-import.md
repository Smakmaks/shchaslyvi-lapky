# Імпорт товарів із RO App API

Цей скрипт отримує товари з вашого акаунту RO App і зберігає їх у `products.json` для використання на статичному сайті.

---

## 1. Встановлення залежностей

```bash
npm install
```

---

## 2. Створення файлу `.env`

Скопіюйте приклад:

```bash
cp .env.example .env
```

Відкрийте `.env` у текстовому редакторі.

---

## 3. Де взяти новий API-ключ RO App

> ⚠️ Якщо ваш старий ключ був випадково збережений у коді або на GitHub — вважайте його скомпрометованим. Створіть новий.

1. Увійдіть до панелі RO App.
2. Перейдіть у **Налаштування → API** (або **Integrations → API Keys**).
3. Натисніть **Створити новий ключ**.
4. Скопіюйте ключ і вставте у файл `.env`:

```
RO_APP_API_KEY=вставте_сюди_ваш_новий_ключ
```

---

## 4. Запуск імпорту

```bash
npm run fetch:products
```

Скрипт:
- Отримує категорії та товари з API
- Нормалізує дані
- Зберігає результат у `products.json`

---

## 5. Перевірка результату

Відкрийте `products.json`. Ви повинні побачити масив товарів:

```json
[
  {
    "id": "123",
    "sku": "ABC-001",
    "name": "Корм для кішок",
    "category": "Корми",
    "price": 250,
    "currency": "грн",
    "description": "Сухий корм...",
    "stock": 10,
    "availability": "in_stock",
    "image": "https://..."
  }
]
```

Кількість знайдених товарів виводиться в консолі після завершення.

---

## 6. Чому `.env` не можна завантажувати на GitHub

Файл `.env` містить API-ключ — секретний токен, який дає доступ до вашого магазину в RO App.

Якщо ключ потрапить на GitHub (навіть у приватний репозиторій):
- Зловмисники можуть отримати доступ до вашого каталогу, замовлень і даних клієнтів.
- GitHub сканує публічні репозиторії та повідомляє про витоки ключів.

**Правило:** `.env` — тільки на вашому комп'ютері. Ніколи не в репозиторії.

Файл `.gitignore` вже налаштований правильно — `.env` виключений із git.

## 7. Вирішення типових проблем

### ❌ Помилка 401 — Неавторизовано
- Ключ у `.env` неправильний або закінчився.
- Рішення: створіть новий ключ у RO App та оновіть `.env`.

### ❌ Помилка 403 — Доступ заборонено
- Ключ не має прав для читання каталогу.
- Рішення: перевірте налаштування прав ключа в панелі RO App.

### ⚠️ Помилка 429 — Забагато запитів
- Скрипт автоматично чекає та повторює запит.
- Якщо проблема повторюється — зменшіть частоту запуску скрипту.

### ⚠️ Порожній список товарів
- Каталог у RO App порожній.
- Або ключ не має доступу до каталогу.
- `products.json` буде створений із порожнім масивом `[]`.
- Перевірте наявність товарів у панелі RO App.

### ⚠️ Поле `availability: "preorder"` або `"unknown"`
Якщо RO App не повертає залишки (поле `stock` або `quantity` відсутнє в API):
- `availability` встановлюється як `preorder` — товар доступний для замовлення, але кількість невідома.
- `unknown` — API не повернув інформацію про наявність.
- У такому випадку рекомендується вручну оновити `products.json` або додати логіку на сайті для відображення статусу.

---

## 8. Важливо: замовлення через сайт та синхронізація з RO App

> **Замовлення через Telegram-бота або статичний сайт НЕ створюють продаж у RO App автоматично і НЕ списують товар зі складу.**

Це означає:
- Залишки у `products.json` оновлюються тільки коли ви вручну запускаєте `npm run fetch:products`.
- Якщо товар продано в RO App — сайт про це дізнається тільки після наступного імпорту.
- Якщо клієнт замовив товар через сайт — потрібно вручну оформити продаж у RO App.

**Рекомендація:** запускайте `npm run fetch:products` перед кожним оновленням сайту, щоб залишки були актуальними.
