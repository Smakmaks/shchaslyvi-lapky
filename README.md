# Зоомагазин — статичний сайт

Статичний сайт зоомагазину для публікації через GitHub Pages.  
Товари завантажуються з `products.json`. Замовлення — через Telegram.

---

## Файли сайту

| Файл | Призначення |
|------|-------------|
| `index.html` | Головна сторінка сайту |
| `style.css` | Всі стилі |
| `script.js` | Логіка: каталог, пошук, фільтри, Telegram |
| `config.js` | Налаштування магазину (назва, Telegram, телефон тощо) |
| `products.json` | Список товарів (генерується скриптом імпорту) |
| `images/` | Фото товарів і заглушка |
| `tools/fetch-ro-products.js` | Локальний скрипт імпорту з RO App (не публікується в сайт) |

---

## Як змінити назву магазину

Відкрийте `config.js` і змініть значення `shopName`:

```js
shopName: "Мій Зоомагазин",
```

---

## Як змінити Telegram

У `config.js` змініть `telegramUsername` (зі знаком @ або без):

```js
telegramUsername: "my_pet_shop",
```

Ця адреса використовується у всіх кнопках "Замовити в Telegram" на сайті.

---

## Як оновити products.json після нового імпорту

1. Перейдіть у папку проекту в терміналі: `cd zoomagasin`
2. Запустіть імпорт: `npm run fetch:products`
3. Скрипт оновить `products.json`
4. Завантажте оновлений файл на GitHub (або зробіть git commit + push)
5. GitHub Pages автоматично опублікує нову версію за кілька хвилин

> **Увага:** замовлення через сайт не списують залишки в RO App автоматично. Оновлюйте products.json після кожного продажу або за графіком.

## Як додавати фото товарів

Варіант 1 — **зовнішнє посилання** (рекомендовано):
- Вкажіть URL фото товару в RO App — скрипт імпорту підхопить його автоматично в поле `image`.

Варіант 2 — **локальні фото**:
1. Покладіть фото в папку `images/` (наприклад `images/korma-pro-plan.jpg`)
2. Для конкретного товару в `products.json` змініть поле `image`:
   ```json
   "image": "images/korma-pro-plan.jpg"
   ```
3. Завантажте фото разом з оновленим `products.json` на GitHub.

Якщо поле `image` порожнє — показується `images/product-placeholder.jpg`.

## Як запустити сайт локально

**Варіант 1 — VS Code Live Server:**
1. Встановіть розширення Live Server у VS Code
2. Відкрийте папку `zoomagasin` у VS Code
3. Клікніть правою кнопкою на `index.html` → "Open with Live Server"

**Варіант 2 — Python:**
```bash
cd zoomagasin
python3 -m http.server 8080
```
Відкрийте в браузері: `http://localhost:8080`

**Варіант 3 — Node.js:**
```bash
npx serve .
```

> ⚠️ Відкривати `index.html` подвійним кліком (через `file://`) не можна — браузер заблокує завантаження `products.json`.

## Як опублікувати через GitHub Pages

### Перший раз:

1. Створіть безкоштовний акаунт на [github.com](https://github.com)
2. Створіть новий репозиторій (наприклад `zoomagasin`)
3. Завантажте всі файли сайту (крім `node_modules/` і `.env`)
4. Перейдіть у **Settings → Pages**
5. У розділі **Source** оберіть гілку `main`, папку `/ (root)` → Save
6. За кілька хвилин сайт буде доступний за адресою:  
   `https://ваш-username.github.io/zoomagasin/`

### Оновлення після зміни products.json:

```bash
git add products.json
git commit -m "Оновлення каталогу товарів"
git push
```

---

## Важливо: замовлення та залишки

- **Сайт не списує залишки** в RO App автоматично.
- **Замовлення через Telegram потрібно обробляти вручну**: отримали повідомлення → уточнили деталі → оформили продаж у RO App вручну.
- **products.json** — це знімок каталогу на момент імпорту. Він не оновлюється сам.
- **Рекомендація**: запускайте `npm run fetch:products` і оновлюйте GitHub щоразу після змін у каталозі RO App.
